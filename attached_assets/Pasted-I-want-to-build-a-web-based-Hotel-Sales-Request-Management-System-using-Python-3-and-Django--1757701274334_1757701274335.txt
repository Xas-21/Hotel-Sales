I want to build a **web-based Hotel Sales Request Management System** using **Python 3 and Django**. The system should replicate and enhance a comprehensive Tkinter-based system I had before, including all advanced features, dashboards, dynamic forms, alerts, and file attachments.

**Core Modules and Requirements:**

1. **Accounts Database**
- Store companies, governments, and travel agencies.
- Fields: name, account_type (Company/Government/Travel Agency), contact_person, position, phone, email, created_at.
- Prevent duplicates (same company, different contact is allowed).
- Ability to add accounts manually, search, and filter.

2. **Requests Module**
- Request Types: Group Accommodation (10+ rooms), Individual Accommodation (1-9 rooms), Event with Rooms, Event without Rooms, Series Group (multiple dates and rooms per date).
- Form Fields:
    - Request received date
    - Linked account
    - Confirmation number
    - Check-in/out dates with automatic nights calculation
    - Dynamic room entry system (add/remove room types and categories: Superior, Deluxe, Executive, Villa with Garden, Villa with Pool)
    - Occupancy types (Single, Double, Twin)
    - Event/Meeting agendas with timing (start, coffee breaks, lunch, end)
    - Transportation options with vehicle type, number of pax, cost
    - Payment deadlines (offer acceptance, deposit, full payment)
    - Status tracking with conditional cancellation reason field appearing only when cancelled
    - File attachment for agreements
    - Real-time total calculations
- CRUD operations: Add, Edit, Cancel, Clear Form, View Details.

3. **Yearly Agreements**
- Fields: linked account, rate type (Corporate, Travel Agency, Group, Government), agreement dates (start, end, return deadline), status (Draft, Sent, Signed, Expired), file attachment.
- Alerts for approaching deadlines.

4. **Sales Calls / Visits**
- Fields: linked account, visit date, city, address, meeting subject (BRM, Negotiations, Contract Review, etc.), next steps, business potential, detailed notes, follow-up requirements.
- CRUD operations with add/edit/view/cancel options.

5. **Dashboard & Analytics**
- Key metrics: total requests by type, confirmed vs cancelled, paid vs unpaid, revenue tracking, lost business value from cancellations, sales call statistics, company database statistics.
- Visuals: charts for request types, statuses, revenue breakdown, recent activity timeline, alert notifications.
- Calendar view for group arrivals and events.

6. **Advanced Features**
- Dynamic forms: show/hide fields based on request type.
- Real-time calculations: room nights × rate × quantity, transportation costs.
- Alert system: notifications for payment deadlines, agreement expirations, follow-ups.
- File attachments: upload and view agreements and event contracts.
- Series Groups: support multiple arrival/departure dates with multiple room types per date.
- CSV export functionality for all data.
- Search and advanced filtering across all modules.

**Technical & Implementation Details**
- Use **Django models** for all entities (Accounts, Requests, Sales Calls, Agreements, Alerts, Transportation, Series Groups, Event Agendas).
- Admin panel for full data management.
- Views, forms, templates for all CRUD operations.
- Use **Bootstrap 5** for responsive and professional UI.
- Include JavaScript for dynamic add/remove fields, real-time calculations, and conditional fields (e.g., cancellation reason).
- Support file uploads and display.
- Implement error handling and input validation.
- Modular code structure with classes.
- Instructions for creating superuser and running the project on Replit.
- Ready to run in browser for single-user use.

**Deliverables**
- Full Django project: models.py, admin.py, views.py, urls.py, templates, static files for CSS/JS.
- Dashboard templates with charts and calendar integration.
- Fully functional forms with dynamic room/agenda entries, transportation costs, series group support, real-time calculations.
- Alerts system for deadlines.
- CSV export for all modules.
- Sample data included for demonstration.
- Comments throughout the code for maintainability.
